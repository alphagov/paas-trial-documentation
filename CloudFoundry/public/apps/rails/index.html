See the official CF guide: Getting Started Deploying Ruby on Rails Apps. Here are the minimal steps for a basic Rails app:

  1. Follow the setup instructions in Getting Started and review concepts. If you are just getting started learning CloudFoundry, you can use the sandbox space:
  cf target -o sandbox
  1. Exclude files ignored by Git.

  2. Create the application on Cloud Foundry.

    cf push <APP_NAME>

  3. Set up the database.

  4. Set additional environment variables.

    cf set-env <APP_NAME> SECRET_TOKEN `rake secret`
# etc.

  5. SSH into the application, then run rake db:setup.

Your app should now be live at APP_NAME.trial.cf.paas.alphagov.co.uk!

Troubleshooting Asset Precompilation

By default, the Rails buildpack performs asset precompilation during the staging phase. This is fine for most Rails apps, but it won’t work for those which need to connect to services (such as the database) during asset compilation.

There are multiple potential solutions for this. For more advice, see the Cloud Foundry document on the subject.
